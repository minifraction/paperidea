## 核心设计理念

### 1. 领域特定性（非常重要）

**不同领域的论文 Idea 提取思路完全不同，不存在统一公式。**

本工具的设计基于以下特定方向：
- **领域**：数学物理 + 博弈模型（如 Minority Game, Public Goods Game 等 Agent-Based Models）
- **特点**：理论性强、模型名称是核心标识符、数学结构决定方法适用性
- **不适用领域**：实验科学（如生物实验、化学合成）、纯工程应用、需要大量数据集的机器学习领域

**为什么其他领域可能不适用？**
- 计算机科学：方法/算法比模型名称更重要，需要搜索最新 SOTA
- 医学/生物：实验验证是关键，文献调研思路完全不同

如果你要在其他领域使用，需要**重新设计整个提示词系统和搜索策略**。

## 2. 论文处理流程（5轮AI分析 + arXiv搜索）

### 2.1 提取的要点

工具从输入论文中提取以下结构化信息：

| 字段 | 用途 |
|------|------|
| **title** | 报告展示 |
| **abstract** | 报告展示 |
| **problem** | 生成 Idea 时评估背景 |
| **method** | AI推断方法迁移、生成 Idea 时评估可行性 |
| **models** | **arXiv搜索的核心**、语义扩展、生成 Idea |
| **algorithms** | AI推断"同大类不同小类"的方法替换 |
| **limitations** | Type A Idea（直接改进）的候选 |
| **future_work** | Type D Idea（深度延伸）的候选 |
| **contributions** | 生成 Idea 时参考 |

### 2.2 完整流程概览

```
输入论文
    │
    ▼
第0轮：语义扩展模型 ──► 扩展出相似模型、更广概念
    │
    ▼
arXiv搜索 ──► 4个关键词 × 4篇 = 最多16篇论文
    │
    ▼
第1轮：AI筛选相关性 ──► 基于摘要评分，过滤低相关性论文（≥6分保留）
    │
    ▼
第2轮：AI深度分析 ──► 提取每篇论文的模型、方法、核心发现、方法迁移机会
    │
    ▼
第3轮：AI推断方法迁移 ──► Type B1（同模型替代方法）、Type B2（同方法新模型）
    │
    ▼
第4轮：AI生成Idea ──► 基于以上全部信息生成5-8个Idea
```

### 2.3 arXiv 搜索策略（仅搜索模型）

**搜索什么：**
- 原始模型名称
- AI语义扩展的"相似模型"（同级概念，最多3个）
- AI语义扩展的"更广概念"（上层概念，最多2个）

**搜索设置：**
- 每个搜索词返回 **4篇** 论文
- 仅搜索 **近10年** 的论文
- 搜索词加引号确保短语匹配（如 `"minority game"`）
- 最多4个搜索词 × 4篇 = 16篇论文

**为什么不搜索算法/问题/局限性？**
- 在理论领域，问题描述过于抽象，搜索返回大量无关结果，算法搜索同理
- 结论和局限性仅用于生成Idea时评估可行性

### 2.4 多轮AI分析详解

**第0轮：语义扩展模型**
- **输入**：原始论文的模型名称
- **输出**：相似模型、更广概念、相关方法
- **目的**：扩展搜索关键词，找到"同级概念"和"上层概念"

**第1轮：AI筛选相关性（去噪关键）**
- **输入**：原始论文模型+问题，+ 搜索到的论文（标题+摘要前300字）
- **输出**：每篇论文的相关性评分（0-10分）和理由
- **过滤**：评分≥6的保留，<6的丢弃
- **目的**：过滤掉标题匹配但内容不相关的论文

**第2轮：AI深度分析（核心改进）**
- **输入**：筛选后的论文（标题+完整摘要），原始论文模型+算法
- **输出**：每篇论文的结构化信息：
  - `analyzed_model`: 该论文实际研究的具体模型
  - `method_used`: 使用的分析方法
  - `key_finding`: 核心发现（1-2句话）
  - `relevance_level`: 与原始论文的关联度（高/中/低）
  - `method_transfer_opportunity`: 具体的方法迁移建议
- **目的**：让AI真正理解论文内容，明确"这篇论文为什么相关"

**第3轮：AI推断方法迁移**
- **输入**：原始论文的模型、算法、问题、方法
- **输出**：
  - **Type B1**: 同模型替代方法（如用深度学习替代蒙特卡洛）
  - **Type B2**: 同方法新模型（基于数学同构性）
- **目的**：基于原始论文本身推断方法迁移机会

**第4轮：AI生成Idea**
- **输入**：原始论文全部信息 + 第2轮深度分析的论文信息 + 第3轮方法迁移建议
- **输出**：5-8个具体Idea，包含可行性、创新性、发表潜力评估
- **特点**：基于**实际论文内容**而非标题猜测，方法迁移建议更具体

### 2.5 搜索结果的去重与排序

**去重**：按arXiv ID去重，避免同一篇论文被多个关键词重复返回

**排序**：
1. 顶刊优先（Nature/Science/PNAS/Nature Communications等）
2. 同年份内，顶刊排在前面
3. 按年份降序（最新的在前）

## 3. Idea 生成方式及重要性排序

工具生成以下类型的研究延伸 Idea，按**推荐优先级**排序：

### ⭐⭐⭐ Type B1: 同模型，替代方法（最推荐）
- **思路**：保持模型不变，尝试相似/替代的分析方法
- **为什么容易发表**：控制变量，对比实验类，实验成本相对较低
- **依据**：AI推断的数学结构分析 + 第2轮深度分析的相关论文方法
- **示例**：在 Minority Game 上用深度学习替代传统统计物理方法

### ⭐⭐⭐ Type B2: 同方法，新模型（最推荐）
- **思路**：将论文的算法/方法应用到数学同构的其他模型
- **为什么容易发表**：方法迁移类，填补研究空白，结果无论正负都有价值
- **依据**：AI推断的数学同构性分析 + 第2轮深度分析的具体关联
- **示例**：将 Public Goods Game 的分析方法应用到 Minority Game

### ⭐⭐ Type D: 基于 Future Work 的深度延伸
- **思路**：将作者指出的未来工作具体化
- **优点**：作者已经指明方向，降低选题风险
- **注意**：需要验证是否已被他人完成

### ⭐ Type B3: 算法应用到新领域
- **思路**：将核心算法应用到完全不同的任务/领域
- **风险**：跨度太大，可能缺乏理论连贯性

### ⭐ Type C: 组合创新
- **思路**：将论文方法与其他技术结合
- **风险**：组合复杂度较高，可行性评估难度大

### ⚠️ Type A: 直接改进（不推荐，除非有充分理由）
- **思路**：解决论文明确提到的 Limitation
- **风险**：如果容易改进，原作者大概率已经做了
- **建议**：仅当你能深入分析为什么原作者"故意"留下这个 Limitation 时才选择

## 4. 如何自定义（加入你的领域经验）

如果你要在其他领域使用，或想改进当前领域的提示词，可以修改以下文件：

### 4.1 提示词文件（`prompts/` 目录）

**语义扩展相关**：
| 文件 | 用途 | 修改建议 |
|------|------|----------|
| `expand_model_system.txt` | 模型语义扩展的系统设定 | 根据你的领域调整"相似模型"的定义 |
| `expand_model_user.txt` | 模型语义扩展的具体指令 | 修改返回的JSON结构和数量要求 |

**多轮AI分析相关**：
| 文件 | 用途 | 修改建议 |
|------|------|----------|
| `filter_relevance_system.txt` | 相关性筛选的系统设定 | 调整评分标准（当前≥6分保留） |
| `filter_relevance_user.txt` | 相关性筛选的具体指令 | 修改相关性评估的维度 |
| `analyze_papers_system.txt` | 深度分析的系统设定 | 调整需要提取的字段 |
| `analyze_papers_user.txt` | 深度分析的具体指令 | **重要**：修改"方法迁移机会"的评估角度 |

**方法推断和Idea生成**：
| 文件 | 用途 | 修改建议 |
|------|------|----------|
| `infer_methods_system.txt` | 方法推断的系统设定 | 根据你的领域调整方法分类标准 |
| `infer_methods_user.txt` | 方法推断的具体指令 | 修改 Type B1/B2 的分析角度 |
| `extract_structure_system.txt` | 论文结构提取的系统设定 | 调整提取字段的优先级 |
| `extract_structure_user.txt` | 论文结构提取的具体指令 | 修改字段定义（如"模型"在你的领域可能叫"系统"） |
| `generate_ideas_system.txt` | Idea生成的系统设定 | 调整Idea类型的权重和评估标准 |
| `generate_ideas_user.txt` | Idea生成的具体指令 | **最重要的文件**：修改Idea类型的定义、强调你的领域特色、调整可行性评估标准 |

### 4.2 顶刊列表（`prompts/top_journals.txt`）

添加你领域的顶级期刊，格式：
```
期刊全称 | 别名1,别名2,...
```

例如：
```
Physical Review Letters | Phys. Rev. Lett.,PRL
Nature Physics | Nature Physics,Nat Phys
```

## 5. 快速开始

### 安装依赖
```bash
pip install -r requirements.txt
```

### 配置 API
1. 运行程序
2. 在"API配置"页面填入 API Key
3. 选择模型厂商和模型
4. 点击保存

### 分析论文
1. 切换到"论文分析"页面
2. 输入论文（支持PDF文件路径或arXiv链接）
3. 点击"开始分析论文"
4. 在"调试信息"区域查看用于arXiv搜索的关键词
5. 等待多轮AI分析完成（会有状态提示）
6. 查看生成的Idea和详细报告

## 6. 设计哲学

这个工具的设计基于以下观察：

1. **模型是理论物理论文的核心**：在ABM（Agent-Based Model）领域，模型名称（如Minority Game）是核心标识符，搜索模型比搜索问题描述更有效。

2. **方法迁移基于数学同构性**：判断"方法A能否用于模型B"应该基于数学结构分析，而非关键词匹配。

3. **控制变量的对比实验最容易发表**：Type B1（同模型不同方法）是控制变量的天然实验，审稿人容易理解其价值。

4. **AI的价值在于推理，而非搜索**：让AI做它擅长的事（数学同构性分析、方法推荐），而不是简单重复arXiv的关键词搜索。

5. **多轮分析优于单轮**：
   - 第1轮去噪：基于摘要内容而非标题判断相关性
   - 第2轮理解：深入理解论文实际做了什么
   - 第3轮推断：基于原始论文本身推断方法迁移
   - 第4轮生成：基于充分信息生成高质量Idea

6. **可解释性很重要**：不仅告诉用户"这篇论文相关"，还告诉"为什么相关"、"具体怎么迁移方法"。
